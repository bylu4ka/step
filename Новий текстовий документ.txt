Створення локальної гілки
Для того, щоб відколотися від гілки main, потрібно, перебуваючи в ній,
створити нову гілку і перейти на цю нову гілку. Це можна зробити двома
способами:
Спосіб 1. Спочатку створити нову гілку, а потім вручну переключитися на неї.
● Для створення нової гілки пишемо в терміналі git branch new-branch.
new-branch - це ім'я нової гілки.
Для перемикання на нову гілку пишемо в терміналі git checkout
new-branch

Спосіб 2. Створити гілку і переключитися на неї однією командою.
● Пишемо в терміналі git checkout -b new-branch. new-branch - це ім'я нової
Advanced Git
● Для перемикання на нову гілку пишемо в терміналі git checkout
new-branch.
Спосіб 2. Створити гілку і переключитися на неї однією командою.
● Пишемо в терміналі git checkout -b new-branch. new-branch - це ім'я нової
гілки, -b це дефіс та буква b.
Злиття змін з головною гілкою
Для того, щоб смердити в main, або будь-яку іншу гілку, потрібно розуміти
такі поняття як target branch та source branch.
Target branch - це гілка, куди ми додаватимемо наші зміни. Після мерджа Target
branch міститиме новий код.
Source branch - це гілка, звідки ми будемо додавати наші зміни в Target branch.
Після мерджа Source branch не зміниться.
Щоб мерджити гілки локально (на вашому комп'ютері), потрібно
перебувати на гілці, яка є Target branch (у нашому випадку master).
Якщо ви зараз не в ній, то в терміналі необхідно ввести команду git
checkout main.
Далі вводимо команду git merge new-branch (new-branch - це ім'я гілки, яку ми
хочемо смерджити у main).
Мердж з віддаленою гілкою
Щоб злити свою гілку з віддаленою гілкою, необхідно виконати такі дії:
● Для початку нам необхідно запустити свою локальну гілку у віддалений
репозиторій. Для цього після індексації та комміту необхідно в терміналі
виконати команду git push origin new-branch. (new-branch - це ім'я нашої
гілки, яка буде створена після цієї команди у віддаленому репозиторії.
Воно має збігатися з назвою гілки, на якій ми зараз перебуваємо).
● Створити Мердж реквест. Після того, як було створено віддалену гілку,
потрібно зробити мердж реквест.
Перебуваючи на сторінці свого репозиторію на сайті Gitlab, необхідно
знайти кнопку Create merge request та натиснути на неї.
● Смерджити у main
Насамперед після створення мердж реквеста, прокручуємо вниз і
знаходимо все ті ж Target branch та Source branch. Якщо потрібно щось
змінювати, залежно від того, що куди мерджити, то міняємо відразу, тому якщо в
майбутньому ці гілки поміняти, можна втратити дані.
У самому мердж реквесті бачимо таку картину:
Для того, щоб оновити стан локальної гілки master - перебуваючи локально у
гілці main, вводимо команду git pull origin master.
